What is Scilab Toolbox Hackathon?

Scilab Toolbox Hackathon is an online non-stop “Scilab toolbox development” competition initiated by the FOSSEE, IIT Bombay. Scilab Toolbox Hackathon aims at inspiring bright minds to build unique problem-specific toolboxes.

There are thousands of useful and well established open source software in the form of Python routines, R routines, C/C++ code, Fortran code, and so on. The objective of this Hackathon is to bring such software as toolboxes to Scilab users. This will enhance the functionality of Scilab. Anyone who knows Scilab should be able to access such software, once it is available as a Scilab Toolbox.

Who can participate?

Anyone who is well acquainted with the functional aspects of Scilab and Xcos or similar numerical computation software can participate in this hackathon. In this challenge, we invite the participation of students, faculty, research scholars, Scilab experts, professionals, etc, across the globe.

The participation can be through teams. The team size can range from 1 to 4 members in the team. Ideally, a team can consist of two domain experts and two coding experts. Apart from the team members, the teams can optionally have a mentor also.

What is the Scilab toolbox?

A Scilab Toolbox is a collection of functions that solve problems of a particular subject. For example, the FOSSEE Optimization Toolbox, FOT, is a collection of COIN-OR libraries that are written to solve Optimization problems. Creating a toolbox out of such popular and useful open source routines enables their usage from Scilab thereby contributing a great resource to Scilab users globally.

Problem Statement

Build a Scilab toolbox to call an external open source software. This process can be broken down into the following steps.

  1. Selection of open source software:
    1. Initially, identify an open-source software whose functionalities/features you would like to include in Scilab. This software can be of any domain such as sciences, any branch of engineering, etc.
    2. It is important that atleast one of the team members has knowledge about the selected domain.
    3. The code should be available with proper reference links.
  2. Interfacing the chosen software with Scilab:
    1. This task has to be done by a team member who is comfortable with interfacing two or more software components with each other.
    2. This is someone who has some CS/IT expertise, even though, they may be from application domains also.
  3. Verification and Testing of the toolbox:
    1. After the creation of Scilab toolbox, it should be able to give the same answer as the selected software would give.
    2. For the verification, some sufficient number of examples should be demonstrated using the created toolbox.
